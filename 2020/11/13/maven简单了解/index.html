maven简单了解

maven 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理。

要了解maven，首先要从maven的环境配置开始。

Maven 特点

  • 项目设置遵循统一的规则。
  • 任意工程中共享。
  • 依赖管理包括自动更新。
  • 一个庞大且不断增长的库。
  • 可扩展，能够轻松编写 Java 或脚本语言的插件。
  • 只需很少或不需要额外配置即可即时访问新功能。
  • 基于模型的构建 − Maven能够将任意数量的项目构建到预定义的输出类型中，如 JAR，WAR 或基于项目元数据的分发，而不需要在大多数情况下执行任何脚本。
  • 项目信息的一致性站点 − 使用与构建过程相同的元数据，Maven 能够生成一个网站或PDF，包括您要添加的任何文档，并添加到关于项目开发状态的标准报告中。
  • 发布管理和发布单独的输出 − Maven 将不需要额外的配置，就可以与源代码管理系统（如 Subversion 或 Git）集成，并可以基于某个标签管理项目的发布。它也可以将其发布到分发位置供其他项目使用。Maven 能够发布单独的输出，如 JAR，包含其他依赖和文档的归档，或者作为源代码发布。
  • 向后兼容性 − 您可以很轻松的从旧版本 Maven 的多个模块移植到 Maven 3 中。
  • 子项目使用父项目依赖时，正常情况子项目应该继承父项目依赖，无需使用版本号，
  • 并行构建 − 编译的速度能普遍提高20 - 50 %。
  • 更好的错误报告 − Maven 改进了错误报告，它为您提供了 Maven wiki 页面的链接，您可以点击链接查看错误的完整描述。

1.安装maven

下载地址：https://maven.apache.org/download.cgi

选择Files中的Binary zip archive即可下载zip压缩包，解压安装。

2.maven的目录结构

bin:可以执行的文件

boot:支持maven正常运行的jar

conf:配置文件

lib:maven运行时所需要的jar

3.maven配置环境变量

MAVEN_HOME:配置maven的安装路径，例：E:\IDE\apache-maven-3.6.3

Path:maven bin文件夹的路径，例：%MAVEN_HOME%\bin

打开cmd，输入mvn -v，验证maven环境变量是否配置成功。

4.maven仓库

本地仓库: 存在于开发者自己计算机上的某一个位置.存放从中央仓库或者私服上下载的jar。

**私服:**基于一个公司,可以存放从中央仓库下载的jar,并且可以为本公司中的项目所使用到的jar做统一的版本控制。

中央仓库:在互联网上存在的仓库,可以在这个仓库下载所需要的jar,可以将jar存放到本地仓库或者远程仓库。

本地仓库配置为：将本地仓库repository文件夹放到maven的根目录下，打开conf文件夹，配置settings.xml
